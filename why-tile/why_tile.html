なぜ「タイル」が単位なのか

AIの計算はほぼ行列積(GEMM)。行列積は「一度読んだ値を何度も使い回す」構造。 その再利用を活かすために 2Dのブロック=タイル単位で処理する。下の C の橙タイルをクリックしてみて。

C の橙タイルをクリック → それを作るのに必要な A の帯(行)と B の帯(列)が光る。

要素ごと vs タイル(なぜタイルが必要か)

要素ごとに計算タイル(32×32)単位
1回の計算量1 積和32×32×32 ≈ 3万 積和
ロードした値の再利用ほぼ無し(毎回読み直し)各値を約32回再利用
メモリアクセス膨大 → メモリ律速(memory wall)1/32以下 → 計算律速に近づく
命令/制御のオーバヘッド1積和ごとに発生3万積和で1回=ほぼ無視

→ 行列積は「読んだ値を使い回せる」のが命。2Dブロックでまとめて処理すれば、ロード1回で大量の積和が回り、 memory wall / ルーフラインを越えられる。要素ごとだと再利用ゼロで潰れる。

なぜ 2D で、なぜ 32×32 か(因果の鎖)

① AIの計算 = 行列積(GEMM)(Llama推論の8〜9割)
▼
② 行列積は 2D の再利用構造:出力 C[i,j]=Σ A[i,k]·B[k,j]。A の1行は C の1行全部に、B の1列は C の1列全部に使い回される
▼
③ 再利用を活かすには 2D ブロック単位:ブロックを一度ロード→大量の積和→メモリアクセス激減(memory wall回避)
▼
④ ハードは固定サイズの 2D 演算グリッド(シストリックアレイ/FPU)+ SIMD幅32 に合わせる → ブロック=32×32 タイル
▼
⑤ だから「タイル」が計算・データ移動・命令の単位になる

あなたが作ったシストリックアレイは、まさに④の「2D演算グリッドでタイル行列積をする」もの。タイルが単位なのは、AIの計算(行列積)の再利用構造が2Dだから。
