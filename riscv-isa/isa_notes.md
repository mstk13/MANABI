# RISC-V で学ぶ ISA(命令セット)

図つき版(ブラウザ): **[isa.html](https://mstk13.github.io/MANABI/riscv-isa/isa.html)**

## 1. ISA とは
**ISA(命令セットアーキテクチャ)= CPUが理解できる命令の語彙**。ソフトとハードの「契約」。
ソフト(コンパイラ)はこの語彙の命令だけを並べる。同じISAなら、中身の違うCPUでも同じプログラムが動く。

## 2. RISC-V の特徴
- **RISC**:命令を単純に・数を絞る。
- **ロード/ストア・アーキテクチャ**:メモリに触るのは load/store だけ。計算はレジスタ上。
- **レジスタ32本**(x0〜x31, x0は常に0)。
- **固定長(基本32bit)**:デコードが楽=パイプライン化しやすい。
- **モジュラー**:基本 **RV32I** + 拡張(M=乗除算, A=アトミック, F/D=浮動小数点, C=圧縮, V=ベクタ, **Zicond**=条件ゼロ …)。

## 3. 命令フォーマット
32bit を R/I/S/B/U/J 形式に区切る。フィールド位置を揃えてあり解読が速い(例: R形式 = funct7,rs2,rs1,funct3,rd,opcode)。

## 4. 分岐を避ける「道具」(核心)
分岐(if/ジャンプ)は予測ミスでパイプラインを捨てて損(→ [pipeline_notes](../cpu-pipeline/pipeline_notes.md))。
**発想:制御依存(どっちへ飛ぶ?)→ データ依存(どっちの値を選ぶ?)** に変える=**branchless**。飛ばない=予測ミスなし。代償は両方計算するムダ。

`x = cond ? a : b;` の例:
- **① 分岐版**: `beq` で飛ぶ。当たれば速いが外すとフラッシュ。
- **② 基本ISA マスク版**: 条件から全0/全1マスクを作り `(a&m)|(b&~m)`。分岐ゼロ・命令増。
- **③ Zicond版**: `czero.eqz t0,a,cond` / `czero.nez t1,b,cond` / `or x,t0,t1`(3命令・分岐なし)。← ISAが用意する道具。

**トレードオフ**:予測しやすい分岐(例:稀なエラー処理)は普通の分岐が得(当たればタダ・片方しか計算しない)。
予測しにくい分岐(50:50, ランダム)や**実行時間を一定にしたい制御系**では branchless が勝つ。

## 5. ISA は性能にどう効くか(Iron Law)
> **実行時間 = 命令数 × CPI × クロック周期**

ISA は3つ全部に効く:
- **命令数**:M(乗除算1命令)、C(コード小)、V(1命令で多数データ)→ 減る
- **CPI**:単純・固定長でデコードしやすく流れる/分岐回避でミス減 → 下がる
- **クロック周期**:命令が単純=クリティカルパス短い → 速いクロック

RISC哲学=命令を単純化して「CPIを下げ・クロックを上げる」。命令数は増えても総合で速く、設計も単純。
