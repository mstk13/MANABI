Tensix コア内部 — ブロックごと解説

Tenstorrent AIアクセラレータ(Grayskull / Wormhole / Blackhole)の演算コア「Tensix」の中身を、制御プレーン(RISC-V)と演算データパス(Unpack→FPU/SFPU→Pack)に分けて。将来の Tenstorrent HW/RTL 設計を見据えて。

📖 全体データフロー①5つのRISC-V②Unpacker ③SrcA/B④FPU(行列)⑤SFPU(ベクトル) ⑥Dst⑦Packer⑧L1パイプライン同期RTL視点
🧭 一言で：Tensixは「5個の小さなRISC-V(指揮者)」＋「行列/ベクトル演算エンジン(奏者)」の組。 RISC-Vは計算せず命令を撃つだけ。データ本体はRISC-Vを通らず、Unpacker/PackerがL1を直接叩く。 制御は Tensix ISA(RISC-V命令とは別物)で、unpack/math/pack の3スレッドが並列パイプラインを回す。

全体データフロー(タイル単位)

実線=データの流れ / 点線=SFPUがDst↔LRegを往復。データはRISC-Vを通らない(=コアはボトルネックにならない)。

① 5つの Baby RISC-V コア(制御プレーン)

小さなインオーダーRISC-V×5。演算はせず、エンジンへの命令発行とデータ移動の指揮に徹する。1本のcompute kernelを書くと、コンパイラが3バイナリ(TRISC0/1/2)に分割し、物理的に別コアで並列に走らせる。

コア別名役割
BRISCData Movement 0NoC経由のデータ移動・ボード設定。主に書き出し側
NCRISCData Movement 1NoC経由のデータ移動。主に読み込み側
TRISC0UNPACK2つのUnpackerを制御
TRISC1MATHFPU・SFPUへ演算命令を発行
TRISC2PACKPackerを制御
🎼 例え：RISC-Vは指揮者、FPU/SFPUは奏者。指揮者は音を出さず(=計算しない)、タクトを振る(=Tensix命令を撃つ)だけ。

② Unpacker(アンパッカ)× 2

L1のタイルを読み、ハードウェアでフォーマット変換して SrcA/SrcB/Dst に置くDMAエンジン。変換は「gasket」という専用機構で行い、ソフトのデコード処理を不要にする。

  • Unpacker 0 → SrcA と Dst に接続
  • Unpacker 1 → SrcB に接続

役割は、DRAM/L1のフラットなテンソル配置と、コプロセッサが期待するtilize済み内部配置の橋渡し。BFP8 / FP16 / FP32 などの形式差もここで吸収する。

③ SrcA / SrcB レジスタ(行列エンジンの入力)

  • ダブルバッファ構成：各々2バンクを持ち、1バンク=1タイル。計算中に次タイルを裏で充填 → パイプラインを止めない。
  • 精度：現行世代で最大19bitのデータ要素(Dstより低精度)。1タイル=1024要素相当。

④ FPU(行列エンジン / Matrix Engine)

乗算器＋加算器を組み合わせた「FPUセル」の行列。SrcA・SrcBを読み、結果をDstへ蓄積する。各セルが実行できるのは：

  1. 累積ドット積(matmul の本体)
  2. 累積 要素ごと加算
  3. 累積 要素ごと乗算
  4. 要素ごと加算

密行列積・畳み込み・プーリングを担当。使う前に mm_init() 等の初期化が要る。 Math Fidelity(演算忠実度)という概念があり、データ形式に応じて完全精度には最大4回の乗算フェーズを回す — 精度と速度のトレードオフ。

⑤ SFPU(ベクトルエンジン / SIMD FP Unit)

活性化関数・超越関数・要素ごと演算など、行列積以外を担うSIMDユニット。FPUと決定的に違う点：

  • ロード/ストア型：Dstを直接は演算できず、内部の LReg(Wormholeで 32要素×32bit)へロード→計算→Dstへ書き戻す。
  • 32bit入力で計算可能。sigmoid, exp, reciprocal 等の複雑関数をソフト的に組める柔軟性(LLKで実装)。演算のチェーンも可。
🔧 ttsim / LLK テストとの接点：あなたが ~/tt-metal のシミュレータで叩いている LLK テストの多くは、この SFPU 層の演算。

⑥ Dst レジスタ(destination / 共有ワークスペース)

compute API から唯一直接見えるレジスタセットで、カーネルの主戦場。

  • FPUの出力先であり、かつSFPUの入力兼出力。
  • 32bit要素を保持(Srcより高精度)。
  • 容量は構成依存で4〜16タイル。16bitモード＋ダブルバッファ無効=16タイル / 有効=8タイル / 32bitモードは半分。
  • タイル内部はface単位で並ぶ：dst_reg[0:3]=1枚目のface…。32×32タイル=16×16のface F0〜F3(row-major)。

⑦ Packer(パッカ)

Dstの結果を受け取り、gasketでフォーマット変換してL1へ書き戻すDMAエンジン。計算とは別コア(TRISC2)で並行動作するので、mathが次タイルを計算中に前の結果を書き出せる。

⑧ L1 SRAM(1.5MB / コア)

コアのスクラッチパッド(キャッシュではない、明示管理)。中身は入力/出力の Circular Buffer(CB)、5コア分のカーネルコード、scratch、セマフォ領域。

CBは生産側 cb_reserve_back → cb_push_back と消費側 cb_wait_front → cb_pop_front で、データ到着とスペース確保を同期する。

パイプライン同期の実際

Dst獲得 → unpack(L1→Src/Dst) → compute(FPU/SFPU) → commit
     → 入力pop / 出力reserve → packer待ち → pack(Dst→L1) → Dst解放

これで通信(unpack/pack)と計算(math)が重なり、3コアが常に動き続ける。

RTL設計視点のまとめ

観点要点
データパスの主役Unpacker/Packer の gasket(フォーマット変換HW)、SrcA/B↔FPU↔Dst の配線、Dst↔LReg↔SFPU
制御プレーンTRISC0/1/2 が発行する Tensix ISA(RISC-V ISAと別)を single-issue in-order で3スレッド並行
精度の設計判断Src=19bit / Dst=32bit / Math Fidelity のフェーズ数 — 面積・電力・精度のトレードオフがRTLに直結
