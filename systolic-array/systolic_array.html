シストリックアレイ(weight-stationary, 3×3)が積和を貯める様子

「systolic=心臓の拍動」。データが1拍ごとに隣のPEへ送られ、流れる途中で積和が貯まる。 重みは各PEに固定(stationary)。活性は右へ→、部分和は下へ↓。

重み W[r][c](固定) 活性 a[r](→右へ進む) 部分和 psum(↓下へ貯まる) 出力 o[c]
t = 0

この図で押さえること

見るもの意味設計での効き(段数/STA)
対角線が光る(r+c=t)計算は波(wavefront)として斜めに広がる入力を階段状にずらす理由
1つのPEの中身乗算1回+加算1回(w·a を psum に足す)これがクリティカルパス=fmaxを決める
o0→o1→o2 が1拍ずつ遅れて出る右の列ほど活性の到着が1拍遅いfill/drain レイテンシ
定常状態1クロックに1結果が出続けるスループットの源(並列度 N²)

段数を増やす=PE内の乗算と加算をレジスタで分割 → クリティカルパス短縮で fmax↑。代償にレイテンシ↑・面積/電力↑。これが パイプライン段数の力がデータパスで再現する点。
