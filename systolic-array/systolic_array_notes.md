# シストリックアレイ(weight-stationary)— 積和を流しながら貯める

図つき版(ブラウザ): **[systolic_array.html](https://mstk13.github.io/MANABI/systolic-array/systolic_array.html)**
(サイクルごとに活性→・部分和↓が流れ、出力 o0→o1→o2 が1拍ずつ出る様子を再生できる)

AIアクセラレータの心臓。Llama推論の演算8〜9割は GEMM(行列積)で、それを**空間に展開して並列化**するのがシストリックアレイ。
ここでは weight-stationary 型を 3×3 で、`o[c] = Σ_r a[r]·W[r][c]`(各列が重み行列の1列との内積)として追う。

---

## 1. PE(セル)1個の中身

各PEは**重み `w` を1個だけ保持(stationary=動かない)**。毎クロック:

```
        psum_in(上から)
            │
  a_in ───►[ w ]───► a_out      a_out = a_in をそのまま右へ
 (左から)    │
            ▼
        psum_out = psum_in + w * a_in   下へ
```

- 左の活性 `a` を**そのまま右へ**渡す
- 上の部分和に **`w·a` を足して下へ**渡す
- 中身は **乗算1回+加算1回**だけ。→ これがPEのクリティカルパス。

## 2. 流れ:重みは固定、活性は→右、部分和は↓下

- 1つの**列を縦に部分和が降りる間に、その列の積和が完成**する。
- 活性は右へ1拍ずつ進むので、**右の列ほど活性の到着が1拍遅い** → 出力 o0,o1,o2 も1拍ずつ遅れて出る。
- 計算は**対角線の波(wavefront, r+c=t)**として斜めに広がる。だから入力を階段状にずらして投入する。

## 3. 具体例(図と同じ)

W = [[1,2,3],[4,5,6],[7,8,9]]、a = [2,3,4] →
- o0 = 2·1 + 3·4 + 4·7 = **42**
- o1 = 2·2 + 3·5 + 4·8 = **51**
- o2 = 2·3 + 3·6 + 4·9 = **60**

定常状態では**1クロックに1結果**が出続ける。N×N アレイなら **N² 個の積和が空間的に同時進行**=CPUが1個ずつやるのを物理で並列化したのが速さの正体。

---

## 4. 設計への効き(段数 ↔ STA ↔ 物理配置)

- PEのクリティカルパス = **乗算+加算** → これが最小クロック周期=fmax を決める。
- **段数を増やす** = PE内で乗算と加算をレジスタで分割(mult段→add段)。
  → クリティカルパス短縮で **fmax↑**。代償に**レイテンシ(cycle数)↑・面積/電力↑**。
- これは [パイプライン段数](../cpu-pipeline/pipeline_notes.md) の力が**データパスで再現**する点。
- さらに [物理設計](../physical-design/pd_notes.md) で見たように、**配置・配線後はワイヤRCが乗って slack が悪化**する。
  → 「段数を増やせば速くなる」は合成時の話。**配線後で見て初めて本当の fmax** が分かる。

## 5. データフローの種類(再利用戦略)

| 型 | 止めるもの | 向き |
|---|---|---|
| **weight-stationary**(本ノート/TPU型) | 重み | GEMM、重み再利用が大きい推論 |
| output-stationary | 出力(累算先) | 累算を1か所に貯めたいとき |
| row-stationary(Eyeriss) | 行データ | エネルギー最小化 |

→ Llama推論は重みが大きく再利用も効くので weight-stationary が基本路線。
